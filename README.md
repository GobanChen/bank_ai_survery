# 三組問卷平衡分派系統｜設定說明

這套東西由兩個免費服務組成：

- **GitHub Pages**：放 `index.html`（受試者看到的同意 / 入口頁）
- **Google Apps Script + Google Sheet**：後端，負責「下一個人分到哪一組」並記錄分派

受試者流程：開啟首頁 → 勾選同意 → 按「開始作答」→ 後端把人分到目前人數最少的組 → 自動導向該組問卷。

---

## A. 後端設定（Google Apps Script）

1. 到 Google 雲端硬碟新建一個 **Google 試算表**（隨意命名，例如「問卷分派紀錄」）。
2. 在該試算表中點：**擴充功能 → Apps Script**。
3. 把 `Code.gs` 的內容整段貼進去，覆蓋預設的 `myFunction`。
4. 修改最上方「設定區」的三組網址（`SURVEY_URLS` 的 A / B / C），換成你三份問卷的連結（SurveyCake 或 Google 表單皆可）。
5. 點右上角 **部署 → 新增部署作業**：
   - 類型：選 **網頁應用程式**
   - 執行身分：**我（你的帳號）**
   - 誰可以存取：**任何人**
6. 第一次部署會要求授權，依指示同意即可。
7. 部署完成後會得到一條 **`/exec` 結尾的網址**，複製起來。

> 驗證：把 `你的/exec網址?action=stats` 貼到瀏覽器，應看到
> `{"ok":true,"counts":{"A":0,"B":0,"C":0}}`，代表後端正常。

---

## B. 前端設定（GitHub Pages）

1. 打開 `index.html`，把 `APPS_SCRIPT_URL` 那行換成上一步拿到的 `/exec` 網址。
2. 同時把首頁裡標示 **【請填寫】 / TODO** 的地方補上：指導教授、研究生姓名與 email、抽獎辦法。
3. 把 `index.html` 推到一個 GitHub repo（檔名就叫 `index.html`、放在根目錄）。
4. repo 的 **Settings → Pages** → Source 選 `main` 分支、`/ (root)` → 儲存。
5. 等一兩分鐘，會得到網址：`https://你的帳號.github.io/repo名/`。這就是發給受試者的連結。

---

## C. 監控進度

隨時把 `你的/exec網址?action=stats` 貼到瀏覽器，就能看到三組目前各分派了幾人。
試算表裡的 `assignments` 工作表也會逐筆記錄每次分派的時間與組別。

---

## 設計說明與兩個取捨

**為什麼分派放在後端、不在前端用亂數？**
前端亂數（或 SurveyCake 內建隨機）只能做到「每個人各自隨機」，不保證三組人數平均，N≈300 時可能跑出 92/108/100。後端改成「永遠補給目前最少的那一組」，能讓三組盡量等量，對你後續要用組別當外生變項跑 between-group ANOVA（H12/H13）比較有利。

**取捨一：分派以「點下開始」計數，非「填完」計數。**
有人按了開始卻中途離開，仍會佔一個名額，可能讓有效樣本略微失衡。若要更精準，可在問卷最後一頁設法回寫一筆完成紀錄再據以校正；多數情況下，直接看 `assignments` 紀錄、必要時手動微調即可。

**取捨二：為了不在網頁原始碼洩漏三組網址，這版沒有「後端掛掉就前端亂數」的備援。**
好處是別人看網頁原始碼也看不到另外兩組、無法繞過分流；代價是萬一 Apps Script 當天額度用盡或服務異常，受試者會看到「連線失敗」而非自動退到某一組。若你更在意「絕對不能中斷」、可以接受網址被看到，我可以改成在前端保留三組網址並加上亂數備援——告訴我你的偏好即可。
